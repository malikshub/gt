What is GIT?

by Shubham MalikApril 24th, 2021

What is Git? What is Version Control System?

Git is an open source software which helps us to keep track of all the changes made to the source code over a period along with the information associated with the changes made to it such as what were the changes made to it, and who made those changes. We call such software as the Version Control System or Source Control.

But how is this useful for a beginner?

So, when you start your career in information technology, you will learn different technologies, build projects on that technology, and then you will leave those projects behind and start learning a new technology. And the cycle goes on, looking at your old code or project can be as challenging and confusing as looking at someone else's code.

Git can't just help you build a local repository that stores all the changes made in the source code, but also help you add messages (any things which you can refer to later), and git can also help create a online backup for your projects with the help of online repository management systems such as Github, BitBucket, SourceForge and many more.

Beginning with GIT

  1. Install Git if you haven't done it yet from here.
  2. The next step is to start your project with git or you can set up git for an already existing project or just clone a project from Github.
    • Use git init <directory-address> to initialize git to a particular path or directory.
    • Use git clone <repository-link> to clone a project to your local machine from platforms such as Github using https or SSH link.
  3. After creating a file we need to inculde that file for the next commit using git add <file> and git add -A the -A will add all the files in the repository.
    Note: The add command won't save the changes to the git repository until we commit those changes.
  4. To save changes to your local repository use git commit -m "commit message".
  5. After a commit if you want to revert back to the previous commit, use git revert <commit-id>

Some more useful GIT Commands

Branches are one of the most important part of the git universe. Lets say if you need to add a particular feature to your project and you are not sure if you really sure if that feature will work or not, then you can simply create a branch from that point and test those features, if it works then you can merge that branch to your main branch. Branches also allows diferent people to work on same project simultaneously.

  1. Use git branch <branch-name> to create a new branch.
  2. Use git checkout <branch-name> to switch between branches.
  3. Use git merge <branch-name> to merge a branch to your current branch.

Remote Repository for GIT

With the help of platform such as Github, BitBucket and SourceForge, you can take your git repository to a online platform for remote access and to collaboration with yout teammates. Open source platform such as Github also provide services like free web hosting for your static websites.
To create a online repository for your local repository, create a account on one of the above disscused platforms, also you can create a empty repository which we would link to your local repository.

  1. Use git remote add <name> <url> to create a new connection to your remote repository, add a name as a shortcut to use url in other commands.
  2. Use git push <name> <branch-name> to push the branch to remote and if the branch does not exist, this command will create one.
  3. Use git pull <name> to pull the remote's copy of that branch and merge it with local repository.
